What Algorithm Used ?

Step 1: Collect User Input
The system collects the user's project type, design style, and theme preference.

Step 2: Validate Input
The selected options are checked to ensure valid values are provided.

Step 3: Generate Palette Key
A unique palette key is created by combining the selected style and theme.

Step 4: Match Palette
The system searches the predefined palette database and finds the closest matching palette.

Step 5: Retrieve Colors
The corresponding color set is retrieved:

  • Primary Color
  • Secondary Color
  • Accent Color
  • Background Color
  • Text Color

Step 6: Display Results
The palette is displayed as color cards with HEX values.

Step 7: Enable User Actions
Users can copy HEX codes and export palette variables.

Step 8: Update Interface
The interface updates dynamically without refreshing the page.
